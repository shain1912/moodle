# KODE LMS — 강의영상 R2 업로드 & 로그인 가이드 (AI 작업용)

이 문서는 **다른 AI/작업자**가 이 LMS에 강의 영상을 추가하거나 로그인 흐름을 다룰 때 그대로 따라 할 수 있도록 정리한 운영 가이드다.

- 프로젝트 경로: `H:\moodle`
- 라이브: <https://lms.kodekorea.kr> (= Render `kode-lms`, 같은 서비스)
- 스택: **Node.js + Express**(백엔드) · **Supabase Postgres**(명단/강의/진도 = DB만) · **Cloudflare R2**(영상 파일, S3 호환, egress 무료)
- 핵심 원칙: 브라우저는 DB·R2에 **직접 접근하지 않는다**. 항상 Express API를 거치고, R2는 **서명 URL(presigned)** 로만 접근한다.

## 1. 로그인 방법

세션은 **httpOnly 쿠키**(`lms_session`, JWT, 유효 12h)로 유지된다. 로그인 응답의 `set-cookie`를 받아 이후 요청에 그대로 실어 보내면 된다.

### 1-1. 학생 로그인 — **이름만** (학번 없음)
```
POST /api/auth/student/login
Content-Type: application/json

{ "name": "홍길동" }
```
- 명단(`lms_students`, `active=true`)에서 **공백 무시** 후 이름이 정확히 일치하는 1명을 찾는다.
- 없으면 `401`, 동명이인 2명 이상이면 `409`(이름만으로는 로그인 불가 → 관리자 처리 필요).
- 해당 학생에게 `pin`이 설정돼 있으면 body에 `"pin"`도 같이 보내야 한다(보통 없음).

### 1-2. 관리자(교사) 로그인
```
POST /api/auth/teacher/login
Content-Type: application/json

{ "username": "admin", "password": "147147" }
```
- bcrypt 해시 비교. 성공 시 role=teacher 세션 쿠키 발급.
- 웹에서는 `/admin.html` 에서 업로드·명단·진도 대시보드·CSV 다운로드를 한다.

### 1-3. 기타 인증 엔드포인트
- `POST /api/auth/logout` — 세션 종료
- `GET  /api/auth/me` — 현재 로그인 사용자(`{role, name, ...}`) 확인

### 1-4. 로그인 검증 예시 (Node, UTF-8 안전)
> ⚠️ Windows에서 `curl -d '{"name":"한글"}'` 은 인코딩이 깨질 수 있다. **반드시 Node `fetch`** 로 검증할 것.
```js
const BASE = 'https://lms.kodekorea.kr';
const r = await fetch(`${BASE}/api/auth/student/login`, {
  method: 'POST', headers: { 'content-type': 'application/json' },
  body: JSON.stringify({ name: '홍길동' }),
});
const cookie = (r.headers.get('set-cookie') || '').split(';')[0]; // 이후 요청에 헤더로 재사용
console.log(r.status, await r.json());
```

## 2. 강의 영상 업로드 (R2 + DB 등록)

영상은 **R2에 파일로 저장**하고, 메타데이터(제목/순서/길이/R2 키)는 **`lms_lectures` 테이블**에 한 행으로 등록한다. 둘 다 해야 강의가 보인다.

### 2-1. ⚠️ 반드시 mp4 (H.264 + AAC) 로 올릴 것
브라우저 `<video>`는 **`.mkv`를 재생하지 못한다**(Safari 불가, Chrome도 불안정). 원본이 mkv/mov면 **mp4로 remux** 후 올린다.

코덱이 이미 H.264 비디오 + AAC 오디오면 **재인코딩 없이 컨테이너만 변환**(무손실, 빠름):
```
ffmpeg -y -i "입력.mkv" -c copy -movflags +faststart "출력.mp4"
```
- `-c copy` = 재인코딩 안 함(화질·용량 그대로). `-movflags +faststart` = 웹 스트리밍용으로 moov atom을 앞으로.
- 코덱 확인: `ffmpeg -i "입력.mkv"` 출력의 `Stream ... Video: h264 ... Audio: aac` 확인.
- 코덱이 다르면(예: HEVC/Opus) 재인코딩 필요: `-c:v libx264 -c:a aac` (느림).
- **ffmpeg 위치**: 이 PC엔 PATH에 없음. 번들 바이너리 사용 → `H:\remotion\node_modules\ffmpeg-static\ffmpeg.exe`

### 2-2. 방법 A — 일괄 스크립트 (대량 업로드 권장)
`H:\moodle\_upload_uploads.mjs` 가 레퍼런스 구현이다. **`H:\moodle\uploads\` 폴더에 영상을 넣고** 아래 실행:
```
cd H:\moodle
node _upload_uploads.mjs
```
이 스크립트가 하는 일(새 영상 추가 시 이 패턴을 그대로 재사용):
1. `uploads/` 의 `mkv/mp4/mov/webm` 을 파일명의 `N장` 숫자로 정렬.
2. 각 파일을 `ffmpeg -c copy -movflags +faststart` 로 mp4 remux (`uploads/_mp4/`).
3. `ffmpeg -i` 출력에서 `Duration` 파싱 → 초 단위 길이.
4. R2에 `PutObjectCommand` 로 업로드. **객체 키 = `<uuid>/<안전한파일명>.mp4`** (`ContentType: 'video/mp4'`).
5. `lms_lectures` 에 한 행 insert (`active: true`).
6. 이미 같은 `title`로 등록된 활성 강의는 건너뜀(중복 방지/이어하기).

R2 클라이언트 설정(스크립트 안에 동일하게 있음):
```js
import { S3Client, PutObjectCommand } from '@aws-sdk/client-s3';
const s3 = new S3Client({
  region: 'auto',
  endpoint: `https://${process.env.R2_ACCOUNT_ID}.r2.cloudflarestorage.com`,
  credentials: {
    accessKeyId: process.env.R2_ACCESS_KEY_ID,
    secretAccessKey: process.env.R2_SECRET_ACCESS_KEY,
  },
  requestChecksumCalculation: 'WHEN_REQUIRED',  // R2는 aws-sdk 기본 체크섬과 호환 안 됨 → 반드시 이 옵션
  responseChecksumValidation: 'WHEN_REQUIRED',
});
await s3.send(new PutObjectCommand({
  Bucket: process.env.R2_BUCKET, Key: key, Body: fs.readFileSync(mp4Path), ContentType: 'video/mp4',
}));
```
> 주의: 위 두 `*ChecksumCalculation/Validation: 'WHEN_REQUIRED'` 옵션이 없으면 R2 PUT 이 실패할 수 있다. 서버측 대량 업로드는 **버킷 CORS가 필요 없다**(브라우저가 아니라서).

`lms_lectures` insert 필드:
```js
await supabase.from('lms_lectures').insert({
  title: '1. 오리엔테이션 (OT)',  // 표시 제목 (앞에 번호 붙이는 컨벤션)
  description: '',
  storage_path: key,             // R2 객체 키 ("<uuid>/파일명.mp4")
  duration_seconds: 1592,        // 정수 초 (진도 % 계산 기준)
  size_bytes: 120000000,
  mime_type: 'video/mp4',
  order_index: 0,                // 0부터. 학생 목록 정렬 순서
  active: true,                  // ★ false면 학생/대시보드/CSV에서 숨김
});
```

### 2-3. 방법 B — 웹 관리자 화면 업로드 (1~2개, presigned PUT)
`/admin.html`(관리자 로그인 후)에서 브라우저가 직접 R2에 올린다. 흐름:
1. `POST /api/lectures/upload-url` `{ "filename": "video.mp4" }` → `{ key, url }`(presigned PUT URL) 수신.
2. 브라우저가 그 `url` 로 **`PUT`** (body=파일, `Content-Type: video/mp4`) → R2 직접 업로드.
3. `POST /api/lectures` `{ title, key, durationSeconds, sizeBytes, mimeType }` → DB 등록.
> ⚠️ 방법 B는 브라우저→R2 직접 PUT이라 **R2 버킷 CORS 설정이 반드시 필요**하다(허용 오리진: `https://lms.kodekorea.kr`, `https://kode-lms.onrender.com`, `http://localhost:3000` / 메서드 PUT·GET·HEAD). CORS가 없으면 업로드가 막힌다. 대량은 방법 A를 쓸 것.

### 2-4. 재생 확인
- 학생/교사: `GET /api/lectures/:id/play` → presigned GET `url`(짧은 유효시간, range 지원) + 강의정보 반환. 그 `url`에 `Range: bytes=0-1023` 요청 시 **206**이면 정상.
- 강의 목록: `GET /api/lectures` (로그인 필요) → `active=true` 인 강의만, `order_index` 순.

## 3. 강의 활성/비활성 (숨김)

`lms_lectures.active` 플래그로 노출을 제어한다. **삭제가 아니라 숨김** — `false`면 학생 목록·관리자 대시보드·CSV 모두에서 빠진다. (라우트 3곳이 `.eq('active', true)` 로 필터)

```sql
-- 특정 강의들 숨기기
update public.lms_lectures set active = false where title like '%(구)%';

-- 전부 숨겼다가 다시 살리기
update public.lms_lectures set active = true  where order_index between 0 and 12;

-- 현재 활성 강의 확인
select order_index, title, round(duration_seconds/60.0,1) as 분, active
from public.lms_lectures where active = true order by order_index;
```
> DB만 바꾸면 **즉시 반영**된다(서버 재배포 불필요). 단, 라우트 필터 자체를 바꿨다면 git push → Render 자동 재배포가 필요하다.

## 4. DB 스키마 요약 (참고)

```
lms_lectures (
  id uuid pk,
  title text,             -- 표시 제목
  description text,
  order_index int,        -- 정렬 순서 (0부터)
  storage_path text,      -- R2 객체 키 (★ 이게 R2의 파일을 가리킴)
  duration_seconds numeric,
  size_bytes bigint,
  mime_type text,         -- 'video/mp4'
  active boolean,         -- ★ false=숨김
  created_at timestamptz
)
lms_students(id, name, team, pin, active, ...)   -- 이름만 로그인, active=false면 로그인 차단
lms_progress(student_id, lecture_id, watched_set jsonb, percent, completed, ...)
lms_teachers(id, username, password_hash, ...)   -- bcrypt
```
- Supabase 프로젝트는 **TEAMKODEKOREA**(공유 프로젝트)이고, 충돌 방지를 위해 모든 테이블이 **`lms_` 접두사**다.
- DB 직접 조작은 Supabase 대시보드 SQL Editor 또는 Supabase MCP(`execute_sql` / `apply_migration`, project_id `hpdnfhfpobmrwgxprrhb`)로 한다.
