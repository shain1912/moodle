# 🔑 키 발급 가이드 (그대로 따라 하기)

성호님이 직접 받아야 하는 값은 **총 4개**입니다. 아래 순서대로 복사해서 `.env` 파일에 붙여넣으면 끝납니다.
(나머지 CORS 설정 등은 제가 해드릴 수 있어요. 값만 알려주세요.)

| 받을 값 | 어디서 | `.env` 의 어느 줄에 |
|---|---|---|
| ① Supabase service_role 키 | Supabase 대시보드 | `SUPABASE_SERVICE_ROLE_KEY=` |
| ② R2 Account ID | Cloudflare R2 | `R2_ACCOUNT_ID=` |
| ③ R2 Access Key ID | Cloudflare R2 | `R2_ACCESS_KEY_ID=` |
| ④ R2 Secret Access Key | Cloudflare R2 | `R2_SECRET_ACCESS_KEY=` |

> `.env` 파일 위치: **`H:\moodle\.env`** — 메모장이나 VS Code로 열면 됩니다.

---

## ① Supabase service_role 키 받기

1. 아래 주소로 들어갑니다 (이미 우리 프로젝트로 바로 연결됨):
   👉 **https://supabase.com/dashboard/project/hpdnfhfpobmrwgxprrhb/settings/api-keys**
   - (로그인 필요. 만약 위 링크가 안 열리면: 대시보드 → **TEAMKODEKOREA** 프로젝트 클릭 → 왼쪽 맨아래 **⚙ Project Settings** → **API Keys**)

2. 화면에서 **`service_role`** 이라고 적힌 키를 찾습니다.
   - `secret` / `Reveal` (눈 아이콘) 을 눌러 값 표시 → **Copy**
   - 길게 생긴 값입니다 (예: `eyJhbGciOiJIUzI1NiI...` 또는 `sb_secret_...`).
   - ⚠️ 이 키는 **절대 외부에 공유 금지** (DB 전체 권한).

> 💡 화면에 `service_role` 대신 **"Secret keys"** 만 보이면, 거기서 **Create new secret key** → 나온 `sb_secret_...` 값을 써도 똑같이 작동합니다.

3. 복사한 값을 `.env` 의 이 줄에 붙여넣기:
   ```
   SUPABASE_SERVICE_ROLE_KEY=여기에_붙여넣기
   ```

## ② ③ ④ Cloudflare R2 (영상 저장소) — 처음이면 5분

### 2-1. R2 켜기 (처음 한 번)
1. 👉 **https://dash.cloudflare.com** 로그인
2. 왼쪽 메뉴에서 **R2 Object Storage** (또는 **R2**) 클릭
3. 처음이면 **결제 수단(신용카드) 등록**을 요구합니다.
   - ⚠️ 무료 한도(저장 10GB, 대역폭 무료)가 있어도 **카드 등록은 필수**입니다. 우리 사용량(45시간)은 월 1천 원 안쪽이라 걱정 안 하셔도 됩니다.
   - **Purchase R2 / Enable** 버튼 눌러 활성화.

### 2-2. 버킷 만들기
1. R2 화면에서 **Create bucket** 클릭
2. 버킷 이름: **`lectures`** (정확히 이 이름)
3. Location: **Automatic** (기본) 그대로 → **Create bucket**

### 2-3. Account ID 확인 (= 값 ②)
- R2 메인(Overview) 화면 **오른쪽**에 **Account ID** 가 표시됩니다. → 복사
- (또는 잠시 뒤 토큰 발급 화면의 endpoint 주소 `https://<여기부분>.r2.cloudflarestorage.com` 에서 가운데가 Account ID)
   ```
   R2_ACCOUNT_ID=여기에_붙여넣기
   ```

### 2-4. R2 API 토큰 발급 (= 값 ③ ④)
1. R2 Overview 화면 **오른쪽 위** 의 **{ } API** → **Manage R2 API Tokens**
   (또는 "Manage R2 API Tokens" 링크) 클릭
2. **Create API Token** (Create Account API token) 클릭
3. 설정:
   - **Token name**: 아무거나 (예: `kode-lms`)
   - **Permissions**: **Object Read & Write** 선택 ✅
   - **Specify bucket(s)**: *Apply to specific buckets only* → **`lectures`** 선택 (보안상 권장, 선택사항)
   - TTL: 그대로(Forever) 둬도 됨
4. **Create API Token** 클릭
5. 🚨 **다음 화면의 값 2개를 바로 복사하세요. 이 화면을 닫으면 Secret은 다시 못 봅니다!**
   - **Access Key ID** → 값 ③
   - **Secret Access Key** → 값 ④
   ```
   R2_ACCESS_KEY_ID=여기에_붙여넣기
   R2_SECRET_ACCESS_KEY=여기에_붙여넣기
   ```
   - (이 화면 아래쪽 **Endpoint** = `https://<ACCOUNT_ID>.r2.cloudflarestorage.com` — ② Account ID 아직 안 적었으면 여기서 확인 가능)

## ⑤ CORS 설정 (제가 해드릴게요 — 안 하면 업로드 막힘)

이건 좀 헷갈리는 부분이라, **위 4개 값만 주시면 제가 처리**하겠습니다.
직접 하시려면:
1. R2 → **`lectures`** 버킷 클릭 → **Settings** 탭
2. **CORS Policy** → **Add CORS policy** (또는 Edit)
3. 아래 JSON 붙여넣고 **Save**:
   ```json
   [
     {
       "AllowedOrigins": ["http://localhost:3000"],
       "AllowedMethods": ["PUT", "GET"],
       "AllowedHeaders": ["*"],
       "ExposeHeaders": ["ETag"],
       "MaxAgeSeconds": 3600
     }
   ]
   ```

## ✅ 다 했으면

`.env` 파일이 이렇게 채워져 있어야 합니다 (값은 예시):
```
SUPABASE_URL=https://hpdnfhfpobmrwgxprrhb.supabase.co      ← 이미 채워져 있음
SUPABASE_SERVICE_ROLE_KEY=eyJhbGci...                       ← ① 붙여넣기
R2_ACCOUNT_ID=a1b2c3d4e5...                                 ← ② 붙여넣기
R2_ACCESS_KEY_ID=f6g7h8...                                  ← ③ 붙여넣기
R2_SECRET_ACCESS_KEY=i9j0k1...                              ← ④ 붙여넣기
R2_BUCKET=lectures                                          ← 이미 채워져 있음
```

그다음 터미널에서:
```
npm start
```
→ 관리자: http://localhost:3000/admin.html (아이디 `admin`, 비번은 `.env`의 `ADMIN_PASSWORD`)

## 🙋 더 쉽게: 그냥 저한테 맡기기

위 4개 값을 여기 채팅에 붙여넣거나, **`H:\moodle\_secrets.txt`** 파일에 적어두시면
제가 `.env` 기입 + CORS 설정 + 실제 업로드/재생/진도까지 **라이브로 검증**해 드립니다.
(끝나면 그 파일은 삭제하고, 토큰은 보안상 나중에 재발급(rotate) 권장)

`_secrets.txt` 형식 예:
```
SUPABASE_SERVICE_ROLE_KEY=...
R2_ACCOUNT_ID=...
R2_ACCESS_KEY_ID=...
R2_SECRET_ACCESS_KEY=...
```
